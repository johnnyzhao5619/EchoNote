### Task 7: RecordingPanel.tsx — full settings persistence

**Files:**
- Modify: `src/components/recording/RecordingPanel.tsx`

**Background:** The current component reads only 3 of 6 settings on mount and never saves changes back. The new design uses an `initialized` **state** variable (not a `useRef`) to correctly handle the race between `getConfig()` (DB read) and `loadDevices()` (CPAL device enumeration). Since state changes trigger React re-renders, all `useEffect` callbacks with `initialized` in their deps list re-evaluate when config arrives — even if devices loaded first. A `useRef` would NOT trigger re-evaluation.

Key behaviors:
- `setInitialized(true)` is always the **last** call in the getConfig callback — this ensures all state setters have been called before effects re-evaluate.
- The device restoration effect depends on `[devices, savedDeviceId]`. If devices loaded before config, `savedDeviceId` is null and the effect picks `defaultDevice`. When config arrives and sets `savedDeviceId`, the effect re-runs and picks the preferred device.
- Save-back effects guard with `if (!initialized) return` — suppresses spurious saves before config has loaded.
- `default_language: language === 'auto' ? null : language` — sending `null` to the backend sets `default_language` to `Some(None)`, which clears it to auto-detect. This is correct per the tauri-specta flattening behavior.

- [ ] **Step 1: Add `initialized` and `savedDeviceId` state variables** — open `src/components/recording/RecordingPanel.tsx` and add two new state declarations after the existing state declarations (after the `autoProcess` line):

```tsx
const [initialized, setInitialized] = useState(false)
const [savedDeviceId, setSavedDeviceId] = useState<string | null>(null)
```

- [ ] **Step 2: Replace the existing mount `useEffect`** (the one that calls `commands.getConfig()` and `loadDevices()`) with the new full-read version:

```tsx
// Load defaults from AppConfig on mount
useEffect(() => {
  commands.getConfig().then((r) => {
    if (r.status === 'ok' && r.data) {
      const cfg = r.data
      if (cfg.vad_threshold != null)    setVadThreshold(Math.min(cfg.vad_threshold, 0.015))
      if (cfg.default_language)         setLanguage(cfg.default_language)
      if (cfg.default_recording_mode)   setMode(cfg.default_recording_mode as typeof mode)
      if (cfg.default_target_language)  setTargetLang(cfg.default_target_language)
      if (cfg.auto_llm_on_stop != null) setAutoProcess(cfg.auto_llm_on_stop)
      setSavedDeviceId(cfg.last_used_device_id ?? null)
      setInitialized(true)   // must be last — triggers save-back effects on next render
    }
  })
  loadDevices()
}, [loadDevices])
```

- [ ] **Step 3: Replace the existing device selection `useEffect`** (the one checking `defaultDevice && !deviceId`) with the new device restoration effect:

```tsx
// Device restoration — re-runs if savedDeviceId arrives after devices
useEffect(() => {
  if (!devices.length) return
  const preferred = savedDeviceId ? devices.find(d => d.id === savedDeviceId) : null
  const target = preferred ?? devices.find(d => d.is_default)
  if (target) setDeviceId(target.id)
}, [devices, savedDeviceId])
```

- [ ] **Step 4: Add save-back effects** — add these two effects after the device restoration effect (before the `onConfigChange` effect):

```tsx
// Save all logical settings back to config
useEffect(() => {
  if (!initialized) return
  commands.updateConfig({
    default_language: language === 'auto' ? null : language,
    default_recording_mode: mode,
    default_target_language: targetLang,
    vad_threshold: vadThreshold,
    auto_llm_on_stop: autoProcess,
  })
}, [initialized, language, mode, targetLang, vadThreshold, autoProcess])

// Save device ID — separate effect; only saves when non-empty
useEffect(() => {
  if (!initialized || !deviceId) return
  commands.updateConfig({ last_used_device_id: deviceId })
}, [initialized, deviceId])
```
