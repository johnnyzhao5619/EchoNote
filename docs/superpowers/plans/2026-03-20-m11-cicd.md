# EchoNote v3.0.0 M11: 跨平台 CI/CD + 发布流水线 Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 EchoNote v3.0.0 建立完整的 GitHub Actions CI/CD 流水线，覆盖三平台 PR 检查、自动化发布构建、模型 SHA256 校验工具，并配置好 `tauri.conf.json`、`build.rs`、CHANGELOG.md 及版本一致性检查脚本。

**Architecture:** 三条工作流分工明确：`ci.yml` 在所有分支 PR 做矩阵编译检查（质量门禁）；`release.yml` 在 `v*.*.*` tag 推送时通过 `tauri-apps/tauri-action@v0` 产出三平台可安装包并创建 GitHub Release；`models-checksum.yml` 手动触发，使用 Python 脚本下载并计算模型 SHA256，输出到 Workflow Summary 供发布前填写。版本号单一来源于 `tauri.conf.json`，`.github/scripts/check-version.sh` 在 CI 中强制校验 `Cargo.toml` 与之一致。

**Tech Stack:** GitHub Actions, tauri-apps/tauri-action@v0, Swatinem/rust-cache@v2, Rust toolchain (stable + aarch64-apple-darwin target), Node 20, jq, Python 3, Keep a Changelog 格式

  完整内容如下（逐字复制）：

  ```yaml
  name: CI

  on:
    push:
      branches: ["**"]
    pull_request:
      branches: ["**"]

  jobs:
    check:
      name: Check (${{ matrix.os }})
      runs-on: ${{ matrix.os }}
      strategy:
        fail-fast: true
        matrix:
          os: [ubuntu-22.04, macos-latest, windows-latest]

      steps:
        - name: Checkout
          uses: actions/checkout@v4

        - name: Install system dependencies (Linux)
          if: matrix.os == 'ubuntu-22.04'
          run: |
            sudo apt-get update
            sudo apt-get install -y \
              libwebkit2gtk-4.1-dev \
              libappindicator3-dev \
              librsvg2-dev \
              patchelf \
              libssl-dev \
              libgtk-3-dev \
              libayatana-appindicator3-dev

        - name: Install Rust toolchain
          uses: dtolnay/rust-toolchain@stable
          with:
            components: clippy

        - name: Rust cache
          uses: Swatinem/rust-cache@v2
          with:
            workspaces: "./src-tauri -> target"

        - name: Setup Node 20
          uses: actions/setup-node@v4
          with:
            node-version: "20"
            cache: "npm"

        - name: Install frontend dependencies
          run: npm ci

        - name: Check version consistency
          shell: bash
          run: bash .github/scripts/check-version.sh

        - name: Run Rust tests
          working-directory: src-tauri
          run: cargo test --all-features

        - name: Run Clippy
          working-directory: src-tauri
          run: cargo clippy --all-features -- -D warnings

        - name: TypeScript typecheck
          run: npm run typecheck

        - name: Lint
          run: npm run lint

        - name: Tauri build (compile check only)
          uses: tauri-apps/tauri-action@v0
          env:
            GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          with:
            args: --debug
  ```

  > 注意：`tauri build --debug` 仅验证编译成功，不上传产物。`fail-fast: true` 确保任一平台失败立即停止其余矩阵任务，节省 CI 分钟数。

- [ ] **Step 1: 创建 `.github/workflows/release.yml`**

  完整内容如下：

  ```yaml
  name: Release

  on:
    push:
      tags:
        - "v*.*.*"

  jobs:
    release:
      name: Release (${{ matrix.name }})
      runs-on: ${{ matrix.os }}
      strategy:
        fail-fast: false
        matrix:
          include:
            - os: macos-latest
              args: "--target universal-apple-darwin"
              name: "macos"
            - os: windows-latest
              args: ""
              name: "windows"
            - os: ubuntu-22.04
              args: ""
              name: "linux"

      steps:
        - name: Checkout
          uses: actions/checkout@v4

        - name: Install system dependencies (Linux)
          if: matrix.os == 'ubuntu-22.04'
          run: |
            sudo apt-get update
            sudo apt-get install -y \
              libwebkit2gtk-4.1-dev \
              libappindicator3-dev \
              librsvg2-dev \
              patchelf \
              libssl-dev \
              libgtk-3-dev \
              libayatana-appindicator3-dev

        - name: Install Rust toolchain
          uses: dtolnay/rust-toolchain@stable

        - name: Add aarch64-apple-darwin target (macOS universal)
          if: matrix.os == 'macos-latest'
          run: rustup target add aarch64-apple-darwin

        - name: Rust cache
          uses: Swatinem/rust-cache@v2
          with:
            workspaces: "./src-tauri -> target"

        - name: Setup Node 20
          uses: actions/setup-node@v4
          with:
            node-version: "20"
            cache: "npm"

        - name: Install frontend dependencies
          run: npm ci

        - name: Generate app icons
          run: npx tauri icon src-tauri/icons/icon.png

        - name: Build and release
          uses: tauri-apps/tauri-action@v0
          env:
            GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
            TAURI_SIGNING_PRIVATE_KEY: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY }}
            TAURI_SIGNING_PRIVATE_KEY_PASSWORD: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY_PASSWORD }}
            # macOS 代码签名 + 公证
            APPLE_CERTIFICATE: ${{ secrets.APPLE_CERTIFICATE }}
            APPLE_CERTIFICATE_PASSWORD: ${{ secrets.APPLE_CERTIFICATE_PASSWORD }}
            APPLE_ID: ${{ secrets.APPLE_ID }}
            APPLE_TEAM_ID: ${{ secrets.APPLE_TEAM_ID }}
            APPLE_APP_SPECIFIC_PASSWORD: ${{ secrets.APPLE_APP_SPECIFIC_PASSWORD }}
          with:
            tagName: ${{ github.ref_name }}
            releaseName: "EchoNote ${{ github.ref_name }}"
            releaseBody: |
              See [CHANGELOG.md](https://github.com/${{ github.repository }}/blob/main/CHANGELOG.md) for details.
            releaseDraft: false
            prerelease: false
            args: ${{ matrix.args }}
  ```

  > `fail-fast: false` 用于 release 矩阵：一个平台失败不影响其他平台的产物上传，避免因单平台代码签名问题导致其他平台产物丢失。

### Task 3: GitHub Actions — 模型 SHA256 校验工具（手动触发）

**Files:**
- Create: `.github/workflows/models-checksum.yml`
- Create: `.github/scripts/compute-model-checksums.py`

- [ ] **Step 1: 创建 Python 脚本 `.github/scripts/compute-model-checksums.py`**

  ```python
  #!/usr/bin/env python3
  """
  计算 resources/models.toml 中所有模型的 SHA256。
  用于发布前验证/填写模型注册表中的 sha256 字段。
  输出到 stdout（GitHub Actions Workflow Summary 格式）。
  """

  import hashlib
  import sys
  import urllib.request
  from pathlib import Path

  try:
      import tomllib  # Python 3.11+
  except ImportError:
      try:
          import tomli as tomllib  # fallback
      except ImportError:
          print("ERROR: requires Python 3.11+ or 'pip install tomli'", file=sys.stderr)
          sys.exit(1)

  MODELS_TOML = Path(__file__).parent.parent.parent / "resources" / "models.toml"
  CHUNK_SIZE = 8 * 1024 * 1024  # 8 MB


  def sha256_url(url: str) -> str:
      h = hashlib.sha256()
      req = urllib.request.Request(url, headers={"User-Agent": "EchoNote-SHA256-Checker/1.0"})
      with urllib.request.urlopen(req, timeout=300) as resp:
          total = int(resp.headers.get("Content-Length", 0))
          downloaded = 0
          while chunk := resp.read(CHUNK_SIZE):
              h.update(chunk)
              downloaded += len(chunk)
              if total:
                  pct = downloaded / total * 100
                  print(f"  {downloaded // 1024 // 1024} MB / {total // 1024 // 1024} MB ({pct:.1f}%)", flush=True)
      return h.hexdigest()


  def main():
      if not MODELS_TOML.exists():
          print(f"ERROR: {MODELS_TOML} not found", file=sys.stderr)
          sys.exit(1)

      with open(MODELS_TOML, "rb") as f:
          config = tomllib.load(f)

      models = config.get("models", [])
      if not models:
          print("No models found in models.toml")
          sys.exit(0)

      summary_lines = [
          "## Model SHA256 Checksums",
          "",
          "| Model ID | Expected SHA256 | Computed SHA256 | Match |",
          "|----------|-----------------|-----------------|-------|",
      ]

      all_ok = True
      for m in models:
          model_id = m.get("id", "unknown")
          url = m.get("url", "")
          expected = m.get("sha256", "")
          print(f"\nComputing SHA256 for: {model_id}", flush=True)
          print(f"  URL: {url}", flush=True)
          computed = sha256_url(url)
          match = computed == expected if expected else "N/A (not set)"
          match_icon = "✅" if computed == expected else ("⚠️ NOT SET" if not expected else "❌ MISMATCH")
          if expected and computed != expected:
              all_ok = False
          summary_lines.append(
              f"| `{model_id}` | `{expected or '(not set)'}` | `{computed}` | {match_icon} |"
          )
          print(f"  Computed: {computed}", flush=True)

      summary_lines.append("")
      summary_lines.append(f"**Status:** {'All checksums verified ✅' if all_ok else 'Mismatch detected ❌'}")

      # Write to GitHub Step Summary
      import os
      summary_path = os.environ.get("GITHUB_STEP_SUMMARY")
      if summary_path:
          with open(summary_path, "w") as f:
              f.write("\n".join(summary_lines))
      else:
          print("\n" + "\n".join(summary_lines))

      sys.exit(0 if all_ok else 1)


  if __name__ == "__main__":
      main()
  ```

### Task 4: 版本一致性检查脚本

**Files:**
- Create: `.github/scripts/check-version.sh`

- [ ] **Step 1: 创建 `.github/scripts/check-version.sh`**

  ```bash
  #!/bin/bash
  # 验证 tauri.conf.json 与 Cargo.toml 版本号一致（单一来源：tauri.conf.json）
  # 退出码 0 = 一致，退出码 1 = 不一致
  set -euo pipefail

  TAURI_VERSION=$(jq -r '.version' src-tauri/tauri.conf.json)
  CARGO_VERSION=$(grep '^version' src-tauri/Cargo.toml | head -1 | cut -d'"' -f2)

  echo "tauri.conf.json version : $TAURI_VERSION"
  echo "Cargo.toml version      : $CARGO_VERSION"

  if [ "$TAURI_VERSION" != "$CARGO_VERSION" ]; then
    echo ""
    echo "ERROR: Version mismatch!"
    echo "  tauri.conf.json = $TAURI_VERSION"
    echo "  Cargo.toml      = $CARGO_VERSION"
    echo ""
    echo "Fix: update Cargo.toml [package] version to match tauri.conf.json."
    exit 1
  fi

  echo "OK: versions match ($TAURI_VERSION)"
  ```

  ```markdown
  ## [3.0.0] - 2026-03-20

  > **Breaking Change:** EchoNote v3.0.0 is a complete rewrite using Tauri 2.x + Rust + React 18.
  > Data from previous Python-based versions (v1.x/v2.x) is **not** migrated.

  ### Added

  - **Tauri 2.x + Rust backend**: Complete rewrite replacing PySide6 with a native Tauri desktop application
  - **Real-time audio transcription**: cpal audio capture → rubato resampling → whisper-rs (whisper.cpp) streaming transcription with live subtitle display
  - **Hardware acceleration**: CoreML + Metal on macOS (Whisper + LLM); optional CUDA feature flag for Windows/Linux
  - **Local LLM integration**: llama-cpp-2 (llama.cpp) with streaming token output for summary, translation, and Q&A tasks
  - **Workspace document library**: CRUD, folder hierarchy, full-text search (FTS5), PDF/DOCX import
  - **Local timeline**: Event management without OAuth; link recordings and documents to timeline events
  - **Model download manager**: First-launch guided download with progress bar; SHA256 verification; configuration-driven via `resources/models.toml`
  - **Theme system (VSCode-style)**: 3 built-in themes (Tokyo Night, Tokyo Night Storm, Tokyo Night Light); user-customizable JSON themes with live preview
  - **Internationalization**: Runtime JSON loading for English, Chinese (Simplified), and French
  - **Automatic updates**: Tauri updater with signed releases
  - **Cross-platform installers**: macOS Universal DMG, Windows MSI + NSIS, Linux AppImage + DEB

  ### Changed

  - UI framework: PySide6 → React 18 + TypeScript + Tailwind CSS + shadcn/ui
  - LLM integration: subprocess-based → llama-cpp-2 Rust bindings (streaming, embedded)
  - Translation engine: ONNX → LLM-driven (llama-cpp-2)
  - Database: SQLite via Python → sqlx + SQLite (WAL mode, compile-time SQL verification)
  - Calendar: Google OAuth removed; replaced with local-only timeline

  ### Removed

  - PySide6 / Python runtime dependency
  - Google Calendar OAuth integration
  - ONNX translation engine
  ```

### Task 9: 版本管理说明与发布 SOP

**Files:**
- 无新文件（本 Task 为操作规程文档化，写入 CHANGELOG.md 的开发者注释中）

- [ ] **Step 1: 确认版本单一来源规则**

  版本号定义在 `src-tauri/tauri.conf.json` 的 `version` 字段：
  - `src-tauri/Cargo.toml` 的 `[package] version` 必须与之完全一致
  - `check-version.sh` 在每次 CI 运行时强制校验，不一致则 CI 失败

  修改版本号时，同时修改这两个文件（唯一需手动保持同步的两处）：
  ```
  src-tauri/tauri.conf.json  →  "version": "X.Y.Z"
  src-tauri/Cargo.toml       →  version = "X.Y.Z"
  ```

- [ ] **Step 2: 发布 SOP（Standard Operating Procedure）**

  每次发布按以下顺序操作：

  1. **更新版本号**（两处同步）：
     ```bash
     # 编辑 src-tauri/tauri.conf.json: "version": "3.1.0"
     # 编辑 src-tauri/Cargo.toml: version = "3.1.0"
     ```

  2. **更新 CHANGELOG.md**：
     - 将 `[Unreleased]` 下的条目移动到新版本区块 `## [3.1.0] - YYYY-MM-DD`
     - 更新底部链接

  3. **运行模型 SHA256 校验**（如有新模型）：
     - 手动触发 GitHub Actions `Compute Model Checksums` 工作流
     - 将输出的 SHA256 填入 `resources/models.toml`
     - 提交更新

  4. **创建 release commit 并推送 tag**：
     ```bash
     git add src-tauri/tauri.conf.json src-tauri/Cargo.toml CHANGELOG.md resources/models.toml
     git commit -m "chore: bump version to 3.1.0"
     git tag v3.1.0
     git push origin main --tags
     ```

  5. **等待 `release.yml` 工作流完成**：
     - 三平台产物自动上传到 GitHub Release
     - Release 自动发布（`releaseDraft: false`）

- [ ] **Commit**（无额外文件，此步骤为验证性检查）

  验证 CI 检查脚本在两种场景下的行为：
  ```bash
  # 正常情况（版本一致）→ 退出码 0
  bash .github/scripts/check-version.sh && echo "PASS: exit 0"

  # 异常情况（临时改 Cargo.toml 版本为不同值）→ 退出码 1
  # 验证后记得还原
  ```

## 附录：GitHub Secrets 配置完整清单

发布前必须在仓库 **Settings → Secrets and variables → Actions → Repository secrets** 中配置：

| Secret | 类型 | 获取方式 |
|--------|------|---------|
| `TAURI_SIGNING_PRIVATE_KEY` | 私钥（PEM 文本） | `npx tauri signer generate` 输出 |
| `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` | 字符串 | 生成私钥时自定义设置 |
| `APPLE_CERTIFICATE` | Base64 字符串 | `base64 -i YourCert.p12 \| pbcopy` |
| `APPLE_CERTIFICATE_PASSWORD` | 字符串 | 导出 `.p12` 时设置的密码 |
| `APPLE_ID` | 邮箱字符串 | Apple ID 登录邮箱 |
| `APPLE_TEAM_ID` | 10位字符串 | Apple Developer Portal → Membership |
| `APPLE_APP_SPECIFIC_PASSWORD` | 字符串 | [appleid.apple.com](https://appleid.apple.com) → App-Specific Passwords |

> `GITHUB_TOKEN` 由 GitHub Actions 自动注入，无需手动配置。
